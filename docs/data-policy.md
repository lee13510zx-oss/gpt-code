# koreastudy 데이터 정책

## 무료 운영 원칙

koreastudy는 초기 단계에서 유료 API, 유료 DB, 유료 서버를 사용하지 않는다.

사용자가 많아져 수익이 발생하고 무료 한도 때문에 서비스 품질이 떨어질 때만 유료 인프라 전환을 검토한다.

## 교과서 데이터

- 교과서 원문 전체를 앱에 저장하지 않는다.
- 출판사, 저자, 단원명 같은 메타데이터는 입력받을 수 있다.
- 사용자가 직접 입력한 내용은 브라우저에서 처리하며, 기본 MVP에서는 서버에 저장하지 않는다.
- 사용자는 본인이 권리를 가진 필기, 직접 작성한 정리, 짧은 학습 범위만 입력해야 한다.
- 교과서별 원문 기반 요약은 향후 공식 라이선스, 사용자 소유 자료, 또는 법적으로 허용되는 범위가 확인된 뒤 확장한다.

## 시험지와 모의고사

- 저작권이 있는 시중 문제집, 학교 시험지, 사설 자료를 무단 수집하거나 저장하지 않는다.
- 공개적으로 사용 가능한 자료, 공식 공개 기출, 사용자가 권리를 가진 입력 자료만 활용한다.
- 문제 생성은 원문 복제가 아니라 개념, 유형, 난이도, 풀이 전략을 바탕으로 새 문항을 생성하는 방향으로 설계한다.

## AI 사용

- 기본 동작은 샘플 데이터 기반 fallback이다.
- Gemini 무료 티어 API 키가 있으면 AI 생성을 시도한다.
- 공개 배포에서는 Netlify 환경변수 `GEMINI_API_KEY`와 Netlify Functions를 우선 사용한다.
- 무료 티어에는 사용량 제한과 데이터 처리 조건이 있으므로 민감한 개인정보, 미공개 시험지, 교과서 원문 전체를 입력하지 않는다.

## 업데이트 구조

- 교육과정은 `2015`, `2022`처럼 버전 필드로 관리한다.
- 과목, 단원, 문제 템플릿은 `data/education-data.js`에서 분리 관리한다.
- 나중에 데이터가 커지면 과목별 JSON 파일로 분리한다.
- `data/data-manifest.js`는 현재 단일 파일 구조와 향후 과목군별 분리 계획을 기록한다.
