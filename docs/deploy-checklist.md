# 무료 배포 체크리스트

이 문서는 나중에 사용자가 승인할 수 있을 때 GitHub와 Netlify 무료 플랜으로 배포하기 위한 절차입니다.

## 승인 전 이미 준비된 것

- 정적 웹앱 파일
- `netlify.toml`
- Netlify Functions 폴더
- Gemini API 키를 환경변수로 숨기는 함수 구조
- 키가 없어도 샘플 데이터로 작동하는 fallback
- 배포 후 확인할 README

## 나중에 승인받고 해야 할 일

앱의 `승인` 화면과 `docs/deferred-approvals.md`를 먼저 확인합니다.

1. GitHub 저장소를 만든다.
2. `outputs/koreastudy` 폴더의 내용을 저장소 루트에 올린다.
3. Netlify 무료 플랜에서 GitHub 저장소를 연결한다.
4. Build command는 비워둔다.
5. Publish directory는 저장소 루트로 둔다.
6. Gemini 연결을 사용할 경우 Netlify 환경변수에 `GEMINI_API_KEY`를 등록한다.
7. 배포 후 아래 기능을 확인한다.

## 배포 후 확인 목록

- 홈 화면이 열린다.
- 학교급, 학년, 과목, 단원 선택이 작동한다.
- 요약, 표, 마인드맵, 문제 화면이 작동한다.
- 검증 화면에서 데이터 상태가 표시된다.
- 배포 화면에서 준비됨/승인 필요 항목이 표시된다.
- 학습지 보기와 인쇄/PDF 저장이 작동한다.
- 보관함 저장이 브라우저에서 작동한다.
- 오답노트 저장, 복습 완료, 삭제가 작동한다.
- 품질 화면에서 자가진단 결과가 표시된다.
- Gemini 키가 없을 때도 샘플 데이터로 결과가 나온다.
- Gemini 키가 있을 때 AI 응답이 반영된다.
- 모바일 화면에서 하단 내비게이션이 겹치지 않는다.

## 무료 한도 관련 주의

- Netlify 무료 플랜의 함수 실행 한도를 넘으면 AI 요청이 제한될 수 있다.
- Gemini 무료 티어의 요청/토큰 한도를 넘으면 AI 생성이 실패할 수 있다.
- 실패해도 앱은 샘플 데이터 fallback으로 계속 작동해야 한다.
