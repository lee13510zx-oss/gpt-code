# 나중에 한 번에 승인할 항목

사용자가 지금 승인할 수 없는 상황이므로, 아래 항목은 앱 쪽 구조와 문서만 먼저 만들고 실제 실행은 나중에 승인받는다.

앱 안의 `승인` 화면에서도 같은 범주의 최종 승인 항목을 확인할 수 있다.

## 계정과 배포

- GitHub 저장소 생성 또는 기존 저장소 연결
- Netlify 계정 연결
- Netlify 환경변수 `GEMINI_API_KEY` 등록
- 실제 공개 URL 배포
- 배포 후 Netlify Function 실행 로그 확인
- 배포 후 앱 내 `배포` 화면의 승인 필요 항목을 실제 상태로 갱신
- 로컬 또는 배포 URL에서 브라우저 수동 QA 수행
- 사용자 도메인 연결 여부 결정

## 외부 API와 키

- Gemini API 키 발급 및 입력
- 나이 제한 또는 계정 제한으로 Gemini API 키를 발급할 수 없는 경우 무료 규칙/샘플 기반 모드 유지
- 무료 티어 사용량 초과 시 유료 전환 여부
- 다른 AI API 추가 여부

## 저작권과 콘텐츠

- 교과서 원문 기반 요약 기능의 사용 범위
- 출판사/저자별 교과서 메타데이터의 공식 출처 검증
- 학교 시험지, 시중 문제집, 사설 자료 분석 가능 범위
- 공식 기출 또는 공개 자료의 수집·가공 가능 범위
- 2015 개정과 2022 개정의 과목별 세부 성취기준 공식 대조
- 공개 자료를 앱 내부 데이터로 재가공해도 되는지 확인

## 유료 전환 후보

- 사용자 수 증가 후 AI 유료 한도 확대
- 사용자 계정과 클라우드 저장용 DB
- 공식 콘텐츠 라이선스
- 검색 인덱스와 분석 도구

## 현재 처리 방식

- 승인 전에는 무료 정적 웹앱과 브라우저 저장소만 사용한다.
- 승인 전에는 저작권 있는 원문을 앱에 저장하지 않는다.
- 승인 전에는 공개 배포에 필요한 계정 작업을 하지 않는다.
- 승인 전에는 유료 서비스에 가입하거나 결제를 요구하는 작업을 하지 않는다.
- 무료 범위의 변경사항은 승인 없이 진행하고, 중요한 변경은 구현 로그에 남긴다.
